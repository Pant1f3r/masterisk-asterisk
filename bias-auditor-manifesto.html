The Four Duties— Non-negotiable obligations of any developer shipping a model that touches human lives
DUTY I

Duty to Audit

No model ships without a disparate impact check. The four audits below are the minimum — not the ceiling, the floor.

DUTY II

Duty to Document

Record findings, tolerances, and fixes. If it isn't documented, it never happened. The model card is evidence, not marketing.

DUTY III

Duty to Refuse

If the data is fundamentally toxic from historical discrimination, do not build the system. "Refusal" is a deliverable.

DUTY IV

Duty to Teach

Ethics is a habit. Enforce it in code reviews. Pass the protocol to the next developer. The discipline propagates or it dies.

The Core Four Pre-Shipment Audits— Each audit has a critical question, an evidence target, and a mathematical tolerance threshold
I
Audit 01 — Representation

VOID

Who is missing?

The Void audit detects under-representation in the training corpus. A model cannot perform on what it has never seen. The "null" output for an erased demographic is not neutrality — it is harm by omission.

What to look for

Demographics absent from the training set, leading to disproportionate error rates, missing outputs, or default-to-majority predictions for under-represented groups. The Gender Shades research (Buolamwini, 2018) found a 35% error rate for dark-skinned women in commercial facial recognition — a direct artifact of Void failure.

Tolerance thresholds
StatusRepresentation Ratio (RR)Interpretation
PASSRR ≥ 0.80Subgroup adequately represented relative to baseline population
CAUTION0.50 ≤ RR < 0.80Under-represented — augment data or document limits
FAIL0.10 ≤ RR < 0.50Severely under-represented — model will fail this group
VOIDRR < 0.10Effectively absent — REFUSE to ship until remedied
FORMULA: RR = (% of subgroup in training set) ÷ (% of subgroup in target population)
EXAMPLE: If African Americans are 14% of US population but only 3% of training data → RR = 0.03/0.14 = 0.21 → FAIL

II
Audit 02 — Historical Bias

MIRROR

Is this just history repeating?

The Mirror audit detects historical bias — the model reproducing the prejudices encoded in past human decisions. Amazon's 2018 hiring AI was a Mirror failure: trained on a decade of resumes from a male-dominated tech workforce, it learned to penalize the word "women's" (as in "women's chess club captain").

What to look for

High correlation between model predictions and historical outcomes that were themselves the product of discrimination. Sexist hiring data produces sexist models. Racist lending data produces racist models. The model is a Mirror — the question is what it is reflecting.

Tolerance thresholds — Statistical Parity Difference (SPD)
Status|SPD| RangeInterpretation
PASS|SPD| ≤ 0.05Negligible disparity in positive outcome rates
ACCEPTABLE0.05 < |SPD| ≤ 0.10Within industry tolerance; document and monitor
CAUTION0.10 < |SPD| ≤ 0.20Significant disparity — investigate causes
FAIL|SPD| > 0.20Disparate treatment — fix or refuse
FORMULA: SPD = P(Ŷ=1 | A=unprivileged) − P(Ŷ=1 | A=privileged)
READING: If hiring model selects 60% of male candidates but only 35% of female candidates → SPD = 0.35 − 0.60 = −0.25 → FAIL

III
Audit 03 — Proxy Detection (THE 4/5ths RULE)

TRICKSTER

What are the hidden proxies?

The Trickster audit detects proxy variables — non-protected features that act as stand-ins for protected classes. ZIP code stands in for race. Surname stands in for ethnicity. School attended stands in for class. "Years of continuous employment" stands in for gender (penalizing maternity leave). The Trickster is the bias that knows how to wear a costume.

LEGAL ANCHOR — 29 CFR § 1607.4(D): The EEOC Uniform Guidelines on Employee Selection Procedures (1978) establish the 4/5ths Rule — also called the 80% Rule — as the federal threshold for presumption of adverse impact. A selection rate for any protected group that is less than 80% of the highest-selected group's rate constitutes prima facie evidence of disparate impact.
Tolerance thresholds — Disparate Impact Ratio (DIR) — 4/5ths Rule
StatusDIR RangeInterpretation
PASSDIR ≥ 0.90Strong parity — no adverse impact concern
ACCEPTABLE0.80 ≤ DIR < 0.90Meets EEOC 4/5ths Rule — within safe harbor
FAIL0.65 ≤ DIR < 0.80Adverse impact presumed — burden shifts to defendant
SEVEREDIR < 0.65Substantial disparate impact — litigation-grade evidence
Tolerance thresholds — Proxy correlation (Pearson |r| or Cramér's V)
Status|r| RangeAction Required
CLEAN|r| < 0.20Negligible correlation — feature retains independent signal
CAUTION0.20 ≤ |r| < 0.40Weak proxy — document and monitor downstream impact
PROXY0.40 ≤ |r| < 0.70Likely proxy — strongly consider removal or transformation
SURROGATE|r| ≥ 0.70Functional surrogate — REMOVE the feature
FORMULA: DIR = (Selection Rate of Unprivileged Group) ÷ (Selection Rate of Privileged Group)
COMMON PROXIES TO FLAG: ZIP code, surname patterns, school attended, "gaps in employment," dialect markers, names of professional associations, photo metadata, language proficiency phrasing, "cultural fit" scores, credit-derived features.

IV
Audit 04 — Intersectionality

CROSSROADS

Who falls through the cracks?

The Crossroads audit detects intersectional harm — bias that hides inside the overlap. A model can look fair across "race" and fair across "gender" while devastating Black women. This is the failure Kimberlé Crenshaw named in 1989, and it is the failure most often missed because aggregate metrics smooth it over.

What to look for

Test every two-way intersection (race × gender, race × age, gender × disability, religion × national origin, etc.) and selected three-way intersections (race × gender × age). Apply the 4/5ths rule to each subgroup independently. A model that passes single-axis audits and fails intersectional audits is not a fair model — it is a model that has been audited badly.

Tolerance thresholds — Intersectional Subgroup Analysis
TestThresholdAction
Minimum cell sizen ≥ 30Below 30, results are statistically unreliable — collect more data
Each 2-way intersectionDIR ≥ 0.80Must independently satisfy 4/5ths Rule
Selected 3-way intersectionsDIR ≥ 0.80Required for high-stakes domains: hiring, lending, healthcare, criminal justice
Aggregation rulemin(DIR) across subgroupsReport the WORST subgroup, not the average
RULE: Report the minimum DIR across all subgroups, not the mean. An average is a place to hide. The worst-served intersectional subgroup is the model's true performance ceiling.

⚑ Pre-Shipment Implementation Checklist

1. Run all four audits: VOID, MIRROR, TRICKSTER, CROSSROADS.
2. Verify the floor — confirm no weaker tools were substituted for the sake of velocity or convenience.
3. Check tolerance — is every metric within published thresholds? If not: FIX OR REFUSE.
4. Test the minimum subgroup — report the worst intersectional cell, not the average.
5. Document everything — the data lineage, audit results, fixes applied, and tolerances accepted (with rationale).
6. Codify the "No" — if the system cannot be made fair, document the refusal and the rationale. Refusal is a deliverable.
7. Pass it on — code-review the protocol into the next engineer. Ethics is a habit.

// Integration with GODMODE Reset Protocol

This module exposes a global window.BiasAuditor namespace with pure functions for each audit. Drop this file into your masterisk-asterisk repo and import it before the GODMODE scanner runs, or call the API directly from your existing scanDocument() handler.

// 1. Drop this file into the repo:
//    github.com/Pant1f3r/masterisk-asterisk/public/bias-auditor-manifesto.html

// 2. In your main index.html, include the script logic. Either embed
//    or extract the <script> block below into a separate .js file.

// 3. Call the API from your scanner:
const voidResult     = BiasAuditor.voidAudit({ trainPct: 3.0, populationPct: 14.0 });
const mirrorResult   = BiasAuditor.mirrorAudit({ privRate: 0.60, unprivRate: 0.35 });
const tricksterResult = BiasAuditor.tricksterAudit({ privRate: 0.50, unprivRate: 0.35 });
const crossResult    = BiasAuditor.crossroadsAudit({
  subgroupRate: 0.18, subgroupN: 142,
  referenceRate: 0.55, referenceN: 890
});

// 4. Each result returns:
//    { status: 'PASS'|'CAUTION'|'FAIL'|'CRITICAL', metric: number, message: string, threshold: object }

// 5. Roll the four results into an overall verdict:
const verdict = BiasAuditor.aggregate([voidResult, mirrorResult, tricksterResult, crossResult]);
// → { overallStatus, worstAudit, recommendation }

Note: The aggregate() function applies the Crossroads principle to the audit set itself — the overall verdict is the worst single audit result, not the average. A model fails as severely as its weakest audit.
